PHP

De meeste mensen die iets met web development doen komen ergens in hun carrière wel een keer in aanraking met PHP, niet zo gek aangezien meer dan 81% van de websites waarvan de backend taal bekend is draait op php. PHP staat voor "PHP: Hypertext Preprocessor".

De oorsprong van PHP

PHP staat voor "PHP: Hypertext Preprocessor" , dit is echter niet altijd zo geweest. PHP zag het licht als "Personal Home Page/Forms Interpeter" en werd in 1994 ontwikkeld door Rasmus Lerdorf. In het begin was PHP niet meer dan een set van CGI's die Lerdorf gebruikte om bij te houden wie zijn online cv bezocht. maar in de loop van de tijd schreef hij er steeds meer tools bij en al snel begon het uit te groeien tot de taal die we vandaag kennen als PHP. Met de release van PHP 3 werd de naam van PHP/FI veranderd naar PHP en kreeg het de huidige betekenis, namelijk "PHP: Hypertext Preprocessor" ook was dit de eerste versie die echt lijkt op de PHP zoals we die vandaag kennen.

PHP structuur

PHP is een serverside scripting language, dit betekent dat de PHP code nooit in de browser te zien is. Een pagina die geschreven is in PHP bevat een mix van HTML en PHP en als de pagina wordt opgevraagd dan wordt de PHP code door de server uitgevoerd wat resulteert is stukken HTML code die vervolgens de PHP code in de opgevraagde pagina vervangt. De server bevat bijvoorbeeld de volgende PHP pagina: <!Doctype html> <HTML> <body> <?php echo 'some text'; ?> </body> </HTML> Als deze pagina door een browser wordt opgevraagd dan wordt de PHP code door de server uitgevoerd en is de resulterende code voor de browser: <!Doctype html> <HTML> <body> some text </body> </HTML> Op deze manier is het eenvoudig om dynamische stukken in een website te bouwen. een bijkomend voordeel is dat men de PHP code waar je als developer hard op hebt zitten zwoegen niet makkelijk kan stelen omdat deze normaal gesproken nooit de server verlaat.

PHP 7

Met de introductie van PHP 7 zijn er een hoop mogelijkheden toegevoegd aan de taal. Object georiënteerd programmeren is al een lange tijd mogelijk in PHP maar vreemd genoeg zijn de volgende functies pas in PHP 7 toegevoegd.

  • ?? de suiker voor if(a!=null)then a else b
  • Integer division
  • return type definitions
  • scalar definitions
  • volledige support voor unicode

Een andere toevoegingen betreft de <=> operator, deze doet een vergelijking en geeft -1, 0 of 1 terug als a < b, a = b of a > b. Verder bevat versie 7 van PHP een grote hoeveelheid syntactische suiker. voor de mensen die het niet weten, met syntactische suiker worden dingen als de ?? operator bedoeld. Syntactische suiker maakt niet zozeer nieuwe dingen mogelijk maar maakt het gewoon makkelijker om het op te schrijven.
